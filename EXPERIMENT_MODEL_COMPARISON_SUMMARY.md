# GPT-2 vs TinyLlama Pilot Comparison Summary

This report compares the GPT-2 pilot experiment and the TinyLlama pilot experiment for Experiment 1: KV Importance / Hot-Cold Stability Analysis.

The goal is to summarize pilot observations only. The results below are from a synthetic pilot setting and do not yet imply general LLM behavior.

## 1. Experiment Setting Comparison

Both model runs used the same main analysis script and the same synthetic 40-sample dataset:

```text
analyze_kv_importance.py
data/samples_pilot_40.jsonl
```

Both runs used the same main analysis configuration:

```text
chunk_size: 64
sink_size: 4
recent_window: 128
top_k: 3
max_input_tokens: 900
max_new_tokens: 16
device: cuda
```

The comparison uses summary outputs generated by:

```text
plot_pilot_summary.py
```

Output directories:

| model | output directory |
|---|---|
| GPT-2 | `outputs/kv_importance_gpt2_pilot_40/` |
| TinyLlama | `outputs/kv_importance_tinyllama_pilot_40/` |

Dataset subtype distribution:

| subtype | count | analysis type |
|---|---:|---|
| local | 10 | local |
| early_recall | 10 | long_range |
| middle_recall | 10 | long_range |
| comparison | 10 | long_range |

Important run differences:

| item | GPT-2 | TinyLlama |
|---|---:|---:|
| attention rows | 640 | 556 |

| chunk importance rows | 7672 | 7993 |
| truncated samples | 0 / 40 | 7 / 40 |

Notes:

- GPT-2 completed all 40 samples with 16 decode steps each.
- TinyLlama produced 556 decode-step rows instead of 640 because some samples reached EOS before `max_new_tokens=16`.
- TinyLlama tokenization made 7 `comparison` samples slightly longer than 900 tokens, so they were truncated by 1-6 tokens using `long_range:preserve_prefix_middle_tail`.
- Because tokenizer behavior differs between GPT-2 and TinyLlama, the same text maps to different token lengths and chunk IDs.

## 6. Shared Observations

The following observations are shared across both model pilots, scoped to this synthetic pilot setting:

1. Both models completed the pilot workflow using the same main analysis logic.
2. Both models produced nonzero Cold KV attention mass.
3. Both models showed frequent top-k hot chunks at chunk 0.
4. Both models showed frequent top-k hot chunks in late prompt chunks.
5. Both models had generally high hot-set overlap.
6. For both models, `local` had the highest `recent_mass_mean` among subtypes.
7. For both models, `early_recall` had the highest `sink_mass_mean` among subtypes.

These shared observations do not yet imply general LLM behavior; they indicate patterns worth checking in a broader setting.

## 7. Model Differences

The following model differences were observed in this synthetic pilot setting:

1. GPT-2 placed more average mass on Recent KV, while TinyLlama placed more average mass on Sink KV.
2. GPT-2 had slightly higher overall Cold KV mass than TinyLlama.
3. The highest-Cold subtype differed:
   - GPT-2: `early_recall`
   - TinyLlama: `middle_recall`
4. TinyLlama had lower average hot-set overlap and higher churn than GPT-2 for three subtypes.
5. GPT-2 completed all 640 possible decode rows, while TinyLlama produced 556 rows because some samples generated EOS early.
6. TinyLlama tokenization produced longer prompt lengths, causing 7 small truncations under `max_input_tokens=900`.
7. TinyLlama's top-k chunk frequency showed a stronger middle chunk signal for `middle_recall`.

These differences may reflect model architecture, tokenizer behavior, prompt length differences, EOS behavior, or synthetic dataset structure. They should not be treated as final conclusions without additional controlled runs.

## 8. Preliminary Implications for Hot/Cold KV Stability

In this synthetic pilot setting, both models support the idea that Sink and Recent KV are important regions, but they also show nonzero Cold KV attention mass.

Pilot implications:

- A fixed Sink + Recent retention policy may capture a large portion of attention mass in these runs.
- Cold KV is not negligible in either model, especially for recall-style subtypes.
- The top-k hot set is mostly stable but not perfectly fixed.
- TinyLlama shows somewhat more hot-set churn than GPT-2 in this pilot.
- Request subtype appears useful for organizing observations, but stronger evidence would require more varied data and additional models.

This does not yet imply that a dynamic retrieval system is required. It only suggests that Cold KV behavior is worth tracking in follow-up experiments.

## 9. Limitations and Next Steps

Limitations:

- The dataset is synthetic and template-generated.
- Only two small models were compared.
- GPT-2 and TinyLlama use different tokenizers, so token counts and chunk IDs are not directly identical.
- TinyLlama had 7 small truncations under the 900-token input budget.
- TinyLlama produced fewer decode steps because some samples reached EOS early.
- Attention mass is an importance proxy, not a causal measurement.
- The current analysis averages over layers and heads.
- The current pilot does not evaluate answer correctness.
- Chunk frequency is influenced by prompt template, tokenizer, chunk size, and max input length.

Recommended next steps:

1. Inspect GPT-2 and TinyLlama summary plots side by side.
2. Consider rerunning TinyLlama with a slightly larger `max_input_tokens` if avoiding truncation is important.
3. Decide whether early EOS should be accepted as part of TinyLlama behavior or controlled through prompt formatting.
4. Keep `analyze_kv_importance.py` stable unless a correctness issue is found.
5. If the current pilot observations look reasonable, run a larger or more naturalistic dataset before moving to larger target models.
6. For future analysis, consider adding optional layer/head breakdowns as a separate extension rather than changing the current main logic.

# GPT解釋

可以，這份比較結果其實已經可以形成一個**完整的 pilot observation** 了。先講總結：

**GPT-2 和 TinyLlama 都顯示 Sink / Recent 是主要 Hot KV 區域，但 Cold KV 不是 0；而且 TinyLlama 比 GPT-2 更顯示出「某些 middle Cold chunks 會因任務類型變重要」的跡象。**

## 1. 實驗設定是否公平？

大致公平。

兩個模型都用同一份 40 筆 synthetic dataset、同一個 `analyze_kv_importance.py`、同樣的 `chunk_size=64`、`sink_size=4`、`recent_window=128`、`top_k=3`、`max_input_tokens=900` 和 `max_new_tokens=16`。

但有兩個差異要注意：

```text
GPT-2：640 rows，全部 sample 都跑滿 16 decode steps
TinyLlama：556 rows，部分 sample 提前產生 EOS
```

另外 TinyLlama 有 7 筆 comparison samples 被截斷 1～6 tokens。

所以你可以比較兩者趨勢，但要記得說：

```text
因為 tokenizer、EOS 行為和 token chunk 邊界不同，兩個模型的 chunk_id 不能完全一對一比較。
```

---

## 2. Overall Attention Mass：兩個模型看哪裡？

整體結果是：

```text
GPT-2:
Sink    0.2871
Recent  0.5340
Cold    0.1789

TinyLlama:
Sink    0.4949
Recent  0.3467
Cold    0.1584
```

這代表很重要的一件事：

```text
GPT-2 比較偏向看 Recent KV；
TinyLlama 比較偏向看 Sink / 開頭 KV。
```

但是兩個模型共同點是：

```text
Cold KV 都不是 0。
```

GPT-2 的 Cold KV mean 是 0.1789，TinyLlama 是 0.1584，兩者都代表中間區域仍然拿到一部分 attention。報告也明確整理，GPT-2 最大 attention mass 在 Recent KV，而 TinyLlama 最大 attention mass 在 Sink KV，但兩者 Cold KV 都非零。

所以這裡不能說：

```text
Cold KV 沒用。
```

比較正確是：

```text
Cold KV 平均比 Sink / Recent 低，但仍有穩定的注意力。
```

---

## 3. Subtype 結果：不同問題類型真的有差

比較重要的 subtype 結果是：

```text
GPT-2:
local 的 Recent 最高
early_recall 的 Cold 最高
early_recall 的 Sink 最高

TinyLlama:
local 的 Recent 最高
middle_recall 的 Cold 最高
early_recall 的 Sink 最高
```

報告裡也指出，TinyLlama 在所有 subtype 上都比 GPT-2 有更高 Sink KV mass、更低 Recent KV mass；Cold KV 大致同一個量級，但「哪一類 request 的 Cold 最高」不一樣。

白話來說：

```text
local：答案在最後，所以 Recent KV 比較重要。
early_recall：答案在開頭，所以 Sink / early region 比較重要。
middle_recall：答案在中間，所以理論上 Cold KV 會比較重要。
```

TinyLlama 的結果很符合這個預期，因為 `middle_recall` 的 Cold KV 最高。GPT-2 則是 `early_recall` 的 Cold 最高，可能跟 GPT-2 的 tokenizer、模型能力或 synthetic prompt pattern 有關。

這個結果可以拿來說：

```text
request type 會影響 KV importance 分布。
```

但要保守說：

```text
這是在 synthetic pilot setting 中觀察到的趨勢。
```

整體 hot-set overlap 是：


churn rate 是：

意思是：

所以你可以回答老師的問題：

目前 pilot 的答案是：

```text
大部分時間 hot chunks 是穩定的，但不是完全固定。
```

這句很重要，因為它同時支持兩件事：

```text

```

## 7. 目前可以形成的核心說法

你現在可以把這個實驗整理成這樣：

```text
在 GPT-2 和 TinyLlama 的 synthetic pilot setting 中，Sink / early region 和 Recent KV 是主要 attention 區域，top-k hot chunks 大多集中在 chunk 0 和 late chunks。因此，固定保留 Sink + Recent KV 可以作為合理 baseline。

```

這句話就很適合放到報告或跟老師討論。
